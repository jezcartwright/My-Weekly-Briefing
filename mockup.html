Briefing Library — widen .wk cards to match .lh topic-card width

Library cards (.wk) currently sit with an extra 12px horizontal margin inside the scroll area, making them 12px narrower per side than the briefing topic cards (.lh). Removing that margin lines the library cards up with the same edges as the topic cards on every other view.

Card horizontal marginInset from screen edge
Topic card .lh (target)016 px (from .sa padding alone)
Library card .wk — current12 px each side28 px (16 .sa + 12 .wk)
Library card .wk — proposed016 px (flush with .lh)

For comparison, every platform below shows three frames: a reference .lh topic card (top) showing the target edges, the current library state (middle, with dashed orange guides at the target edges), and the proposed library state (bottom) flush with those guides.
